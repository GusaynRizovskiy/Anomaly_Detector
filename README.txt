🛡️ Anomaly Sniffer: Система обнаружения сетевых атак
Anomaly Sniffer — это программный комплекс для мониторинга сетевого трафика и обнаружения аномалий (компьютерных атак) с использованием нейронной сети архитектуры CNN-LSTM Autoencoder.

🚀 Основные возможности
Сбор данных: Захват трафика и агрегация метрик в CSV-файлы.
Обучение: Создание модели "нормального" поведения сети на основе собранных данных.
Валидация: Оценка качества работы модели (Precision, Recall, ROC AUC) на размеченных данных.
Детектирование: Обнаружение аномалий в режиме реального времени (Online) или анализ дампов (Offline).

🛠️ Установка и зависимости
Для работы требуются Python 3.8+ и следующие библиотеки:
scapy — захват и анализ пакетов.tensorflow — работа с нейронной сетью.
pandas, numpy — обработка данных.
scikit-learn — нормализация и расчет метрик.

📖 Режимы работы и использование.
1. Сбор трафика (collect)
 Используется для формирования обучающей выборки (нормальный трафик) или тестовых наборов.
 main.py collect --interface "eth0" --network 192.168.1.0/24 --interval 2 --data-file normal_traffic.csv
--interface: Название сетевого адаптера для прослушивания.--network: Подсеть в формате CIDR, трафик которой нужно учитывать.--interval: Окно агрегации данных в секундах (по умолчанию 10).

2. Обучение модели (train)
Создает модель, которая "запоминает" характеристики нормального трафика.
 main.py train --data-file normal_traffic.csv --epochs 50 --time-step 10 --model-dir models
--time-step: Длина временной последовательности для LSTM.--model-dir: Папка, куда сохранятся model.keras и scaler.pkl.

3. Валидация и оценка качества (validate)
Режим для расчета точности детектора на данных с метками (0 — норма, 1 — атака). Выводит метрики качества и строит графики.
 main.py validate --data-file attack_data_labeled.csv --model-dir models --threshold 0.05
--threshold: Порог ошибки реконструкции (MSE). Если ошибка выше — фиксируется атака.

4. Детектирование в реальном времени (detect-online)
Запускает мониторинг интерфейса. При обнаружении аномалии выводится предупреждение.
main.py detect-online --interface "eth0" --network 10.0.0.0/24 --interval 5

Удаленный мониторинг: Можно отправлять алерты на удаленный сервер:
 main.py detect-online -i eth0 --remote-host 192.168.1.10 --remote-port 9090

📊 Метрики качества (Теория)
Для оценки эффективности датчика в режиме validate используются:
Precision (Точность): Доля реальных атак среди всех срабатываний.
Recall (Полнота): Процент обнаруженных атак от их общего количества.
ROC AUC: Способность модели отличать аномалию от нормы при разных порогах.

📋 Примеры команд для быстрого старта
Сбор данных
python main.py collect --interface "Wi-Fi" --network 192.168.1.0/24 --data-file data.csv
Обучение
python main.py train --data-file data.csv --time-step 10
Детектирование (Offline)
python main.py detect-offline -d test_labeled.csv -m models/model.keras -thr 0.01
Детектирование (Online)
python main.py detect-online --interface "eth0" --interval 2
