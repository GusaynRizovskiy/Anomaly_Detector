🛡️ Anomaly Sniffer: Network IDS Framework
Anomaly Sniffer — это интеллектуальный программный комплекс для обнаружения сетевых атак в реальном времени.
В основе системы лежит нейросетевая архитектура CNN-LSTM Autoencoder, которая обучается на профиле «нормального»
трафика и выявляет аномалии как отклонения от выученного паттерна.

🚀 Ключевые возможности
Интеллектуальный захват:
    Агрегация 26 статистических метрик сетевого трафика (интенсивность, флаги TCP, параметры фрагментации и др.)
Гибридная архитектура:
    Использование сверточных слоев (CNN) для извлечения признаков и рекуррентных слоев (LSTM) для анализа временных зависимостей.
Гибкие режимы:
    Collect:
     Формирование датасетов в формате CSV.
    Train:
     Обучение модели на чистом трафике (Unsupervised Learning).
    Validate:
     Оценка качества (Precision, Recall, F1, ROC AUC) на размеченных данных.
    Detect:
     Мониторинг интерфейса в реальном времени с выводом алертов.

🛠️ Установка и зависимости
Система разработана на Python 3.8+.
Для работы требуются права суперпользователя (root/admin) для доступа к сетевому интерфейсу.
Зависимости:
    scapy — низкоуровневый захват и анализ пакетов.
    tensorflow — обучение и инференс нейронной сети.
    pandas, numpy — обработка и агрегация данных.
    scikit-learn — нормализация (MinMaxScaler) и расчет метрик.
    matplotlib, seaborn — визуализация результатов.

📖 Инструкция по использованию
1. Сбор данных (Режим collect)
    Необходим для создания обучающей выборки. Запускайте в сети без активных атак.

    main.py collect --interface "eth0" --network 192.168.1.0/24 --interval 2 --data-file normal.csv

    --interval: Окно агрегации (в секундах). Чем меньше интервал, тем выше чувствительность к коротким атакам.

2. Обучение модели (Режим train)
    Модель учится восстанавливать нормальный трафик с минимальной ошибкой.

    main.py train --data-file normal.csv --epochs 50 --time-step 10

    После обучения в папке проекта появятся model.keras и scaler.pkl.

3. Валидация и поиск порога (Режим detect-offline)
    Используется для проверки на данных, где заведомо есть атаки (нужны метки 0/1).

    main.py detect-offline --data-file attack_labeled.csv --threshold 0.05

    --threshold: Порог MSE. Подбирается экспериментально для минимизации ложных срабатываний (False Positives).

4. Мониторинг в реальном времени (Режим detect-online)
    Запуск сенсора в режиме защиты.

    main.py detect-online --interface "eth0" --network 192.168.1.0/24 --interval 5

📊 Математическая основа (MSE & Threshold)
Система работает по принципу ошибки реконструкции:
    Автокодировщик сжимает входящие данные и пытается восстановить их обратно.
    На нормальном трафике ошибка восстановления ($MSE$) будет низкой.
    При появлении аномальных паттернов (атаки) модель не сможет точно восстановить данные, и $MSE$ резко возрастет.
    Если $MSE > Threshold$, система генерирует оповещение об атаке.$$MSE = \frac{1}{n} \sum_{i=1}^{n} (x_i - \hat{x}_i)^2$$

📂 Структура проекта
    main.py — точка входа, интерфейс командной строки.
    core/sniffer.py — многопоточный захват пакетов и расчет метрик.
    core/anomaly_detector.py — описание и логика нейронной сети.
    core/data_processor.py — предобработка данных и нормализация.
    core/label_preparer.py — утилита для разметки собранных данных.

Разметка: python main.py --mode label --data_file traffic.csv (получаешь traffic_labeled.csv).

Валидация: python main.py --mode validate --data_file traffic.csv --labels traffic_labeled.csv.
